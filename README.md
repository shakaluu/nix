# System Configuration

Managed with [nix-darwin](https://github.com/nix-darwin/nix-darwin) and [home-manager](https://github.com/nix-community/home-manager).

## Structure

```
flake.nix                        # entry point, defines all hosts via mkDarwinSystem / mkNixosSystem
hosts/
  common/
    home.nix                     # shared home-manager config (all hosts)
    app-themes/
      ghostty-theme              # ghostty colour theme
      zsh-theme                  # oh-my-zsh theme
  <hostname>/
    user.config                  # username and home directory
    configuration.nix            # host-specific system config
.github/
  copilot-instructions.md        # Copilot agent instructions for this repo
.gitignore                       # excludes home-manager generated dirs
```

> `ghostty/` and `git/` in the repo root are generated by home-manager and are excluded via `.gitignore`.

## How it works

- `user.config` is imported once in `flake.nix` and passed to all modules via `specialArgs` as `cfg`.
- The `nix-update` alias in `home.nix` automatically uses the correct rebuild command for the current platform (`darwin-rebuild` on macOS, `nixos-rebuild` on Linux).
- System packages are managed via **Homebrew** (macOS) rather than `environment.systemPackages`.

## Applying changes

```sh
nix-update
```

## Adding a new host

### 1. Create the host directory

```sh
mkdir hosts/<hostname>
```

### 2. Create `hosts/<hostname>/user.config`

```nix
{
  name = "your-username";
  home = "/home/your-username";   # use /Users/... on macOS
}
```

### 3. Create `hosts/<hostname>/configuration.nix`

Use an existing host as a starting point:

```sh
cp hosts/DT-FK9XW5PWMN/configuration.nix hosts/<hostname>/configuration.nix
```

Then adjust:

- `nixpkgs.hostPlatform` — e.g. `"x86_64-linux"` or `"aarch64-linux"`
- `system.stateVersion`
- Any platform-specific settings (the `homebrew` block is macOS-only, remove it for Linux)

The `cfg` argument (username, home dir) is automatically injected from `user.config` — no manual import needed in `configuration.nix`.

### 4. Register the host in `flake.nix`

**macOS host:**

```nix
darwinConfigurations = {
  "DT-FK9XW5PWMN" = mkDarwinSystem "DT-FK9XW5PWMN";
  "<hostname>"    = mkDarwinSystem "<hostname>";     # add this line
};
```

**Linux host:**

Uncomment the `mkNixosSystem` helper and the `nixosConfigurations` block at the bottom of `flake.nix`, then add the host:

```nix
nixosConfigurations = {
  "<hostname>" = mkNixosSystem "<hostname>";
};
};
```

### 5. Apply the configuration on the new machine

```sh
# macOS
sudo darwin-rebuild switch --flake ~/.config#<hostname>

# Linux
sudo nixos-rebuild switch --flake ~/.config#<hostname>
```

After the first switch the `nix-update` alias will be available for subsequent updates.
